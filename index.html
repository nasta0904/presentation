Что такое языки разметки?

Языки разметки используются для аннотирования текста, добавления структуры и семантики к данным. Они играют ключевую роль в веб-разработке, конфигурации приложений и обмене данными между системами.

XML

XML (eXtensible Markup Language) - это язык разметки, который определяет набор правил для кодирования документов в формате, удобном для чтения как человеком, так и машиной.

Основные характеристики:

  • Расширяемость - можно создавать собственные теги
  • Самодокументируемость
  • Используется для обмена данными между системами
  • Поддерживает валидацию через DTD или XML Schema

Пример использования:

<?xml version="1.0" encoding="UTF-8"?>
<book>
  <title>Война и мир</title>
  <author>Лев Толстой</author>
  <year>1869</year>
</book>

HTML

HTML (HyperText Markup Language) - стандартный язык разметки для создания веб-страниц и веб-приложений. Вместе с CSS и JavaScript образует основу современных веб-технологий.

Основные характеристики:

  • Предопределенный набор тегов для структурирования контента
  • Поддержка мультимедийного контента
  • Возможность создания интерактивных элементов через формы
  • Семантическая разметка для улучшения доступности и SEO

Пример использования:

<!DOCTYPE html>
<html>
<head>
  <title>Мой сайт</title>
</head>
<body>
  <h1>Заголовок страницы</h1>
  <p>Это абзац текста.</p>
</body>
</html>

YAML

YAML (YAML Ain't Markup Language) - это человеко-читаемый язык сериализации данных, часто используемый для конфигурационных файлов и обмена данными между языками программирования.

Основные характеристики:

  • Минималистичный синтаксис
  • Использует отступы для структуры (без скобок)
  • Поддержка сложных структур данных
  • Часто используется в DevOps и контейнеризации

Пример использования:

# Конфигурация приложения
app:
  name: "Мое приложение"
  version: 1.0.0
  database:
    host: "localhost"
    port: 5432
  features:
    - "auth"
    - "notifications"

YML

YML - это фактически то же самое, что и YAML. Расширение .yml часто используется вместо .yaml для файлов YAML, особенно в системах, где длина расширения файла ограничена тремя символами.

Основные характеристики:

  • Полная совместимость с YAML
  • Идентичный синтаксис и возможности
  • Часто используется в CI/CD конфигурациях (например, GitHub Actions)
  • Популярен для конфигурации Docker Compose

Пример использования (аналогичен YAML):

# .github/workflows/deploy.yml
name: Deploy to production
on:
  push:
    branches: [ main ]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v2
